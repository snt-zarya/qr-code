# Платёжный QR-код · ГОСТ Р 56042-2014

Веб-приложение для генерации платёжных QR-кодов по стандарту **ГОСТ Р 56042-2014**
(формат `ST00012`). Такой код распознают мобильные приложения российских банков
(Сбербанк, Т-Банк, ВТБ, Альфа-Банк и др.) и автоматически заполняют платёжное поручение.

## Возможности

- 🧾 **Обычные и бюджетные платежи** — обязательные реквизиты + дополнительные
  (сумма, назначение, ИНН/КПП) и блок для налогов/ЖКХ (КБК, ОКТМО, статус плательщика,
  налоговый период, УИН).
- ✅ **Проверка реквизитов** — контрольный ключ расчётного и корреспондентского счёта
  по алгоритму ЦБ РФ, валидация ИНН (10/12 цифр), БИК, формата суммы.
- 🔄 **Живой предпросмотр** QR-кода с подсветкой ошибок.
- 💾 **Автосохранение** введённых данных в `localStorage` браузера.
- ⬇️ **Скачивание** QR в форматах **PNG** и **SVG**, копирование строки `ST00012`.
- 🔒 **Приватность** — все вычисления выполняются в браузере, данные никуда не отправляются.

```bash
npm install      # установка зависимостей
npm run dev      # режим разработки → http://localhost:5173
npm run build    # production-сборка в dist/
npm run preview  # локальный просмотр собранной версии
```

## Формат строки (ГОСТ Р 56042-2014)

```
ST00012|Name=ООО «Ромашка»|PersonalAcc=40702810938000000001|BankName=ПАО СБЕРБАНК|BIC=044525225|CorrespAcc=30101810400000000225|Sum=150000|Purpose=Оплата по счёту
```

| Часть      | Значение                                              |
|------------|-------------------------------------------------------|
| `ST`       | служебный признак (Service Tag)                       |
| `0001`     | версия формата                                        |
| `2`        | кодировка: `1`=Windows-1251, `2`=UTF-8, `3`=KOI8-R    |
| `\|`       | разделитель полей                                     |

**Обязательные поля:** `Name`, `PersonalAcc`, `BankName`, `BIC`, `CorrespAcc`.
Сумма (`Sum`) указывается в **копейках** целым числом.
